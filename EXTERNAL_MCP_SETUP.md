# 外部 MCP 服务器集成方案

## 方案 B：使用外部 MCP 服务器

### 什么是外部 MCP 服务器？

外部 MCP 服务器是独立运行的 MCP 服务，提供标准化的工具接口。你的应用作为 MCP **客户端**连接这些服务器，通过 MCP 协议调用它们提供的工具。

## 两种连接方式

### 方式 1：本地进程式 MCP 服务器（当前方式）
- 通过 `stdio` 通信
- 服务器作为子进程运行
- 适合：自己搭建的 MCP 服务器

### 方式 2：远程 HTTP/SSE MCP 服务器（外部服务器）
- 通过 HTTP/SSE 连接
- 服务器运行在远程地址
- 适合：使用别人提供的 MCP 服务器

## 实现方案

### 方案 1：连接远程 HTTP MCP 服务器

**需要的信息**：
- MCP 服务器 URL（如：`http://localhost:8001` 或 `https://mcp-server.example.com`）
- 认证信息（如果需要）

**实现步骤**：
1. 安装 MCP Python SDK（已有）
2. 创建 MCP 客户端，连接到外部服务器
3. 调用服务器提供的工具

### 方案 2：运行本地 MCP 服务器进程

**需要的信息**：
- MCP 服务器启动命令
- 服务器参数

**实现步骤**：
1. 配置服务器启动命令
2. 客户端通过 stdio 连接
3. 调用工具

## 推荐方案

### 推荐：自己搭建本地 MCP 服务器

**优点**：
- 完全控制
- 可以使用你的 API Key
- 不依赖外部服务
- 数据安全

**实现**：
1. 创建一个独立的 MCP 服务器，提供：
   - `search_places` - 搜索景点
   - `get_place_details` - 获取详情
   - `search_hotels` - 搜索酒店
   - `get_weather` - 获取天气

2. 你的应用作为客户端连接这个服务器

3. LLM 通过 MCP 协议调用这些工具
